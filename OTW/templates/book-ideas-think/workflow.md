# Book Ideas Brainstorming Workflow

「次に何の本を書くか」を決めるためのプロセス定義。

---

## 全体の流れ

```
[1. 発散] → [2. 評価] → [3. 決定] → [4. 深掘りへ移行]
Brainstorm    Evaluate     Decide      → book-contents-think
```

---

## Phase 1: 発散（Brainstorm Mode）

### 目的
とにかくアイデアを出す。質より量。

### やること
1. **自己分析:** 専門、経験、強みを `ideas-hub.md` に記入
2. **発散セッション:** 以下の切り口でアイデアを出す
   - 自分が経験した困難と乗り越え方
   - 人からよく聞かれること
   - 「みんな勘違いしてるな」と思うこと
   - 既存の本への不満・物足りなさ
   - 10年前の自分に教えたいこと
   - 業界の常識を疑う視点
3. **記録:** 出たアイデアを Ideas Pool に追加（評価はしない）

### アイデア発掘の問い
- 「あなたが語れて、他の人が語れないことは？」
- 「同じテーマの本で、言われてないことは？」
- 「読者が本当に知りたいのに、誰も書いてないことは？」
- 「失敗から学んだ、人に伝えたい教訓は？」

### Tips
- 「自分なんかが書いていいのか」は後で考える
- タイトルは仮でOK。キャッチーさは後から
- 10個以上を目標に

---

## Phase 2: 評価（Evaluator Mode）

### 目的
アイデアを客観的に比較し、候補を絞る。

### 評価軸

| 軸 | 問い |
|----|------|
| **Authority** | この本を書く資格があるか？説得力があるか？ |
| **Passion** | 本当に書きたいか？最後まで情熱を持てるか？ |
| **Audience** | 誰が読む？その人たちはどれくらい欲しがる？ |
| **Differentiation** | 既存の本と何が違う？なぜ「あなたが」書く意味がある？ |
| **Feasibility** | 書ききれるか？リサーチ量は現実的か？ |

### やること
1. 有望そうなアイデアを5-7個ピックアップ
2. 各アイデアを評価軸でスコアリング（H/M/L）
3. 比較表を作成
4. 上位3つを Shortlist に

### 注意
- 市場規模だけで判断しない（熱量の高いニッチ > 関心の薄いマス）
- 「書ける」より「書きたい」を重視（書けても書きたくないと続かない）

---

## Phase 3: 決定（Decision Mode）

### 目的
「次に書く本」を1つ決める。

### やること
1. Shortlist のアイデアを最終比較
2. 決定基準を明確化（今、何を優先するか？）
   - ブランディング目的なら → Authority, Differentiation重視
   - 収益目的なら → Audience重視
   - 自己表現なら → Passion重視
3. 決断し、Decision Log に記録
4. 選ばなかったアイデアは Parking Lot へ

### 決定を助ける問い
- 「5年後、どの本を書いておけばよかったと後悔する？」
- 「どれなら、今すぐ書き始めたいと思える？」
- 「どの本が出たら、一番誇らしい？」
- 「この本を書くことで、自分は何者になれる？」

---

## Phase 4: 移行

### やること
1. 決定したアイデアの概要をコピー
2. `book-contents-think` に新しいプロジェクトとして持っていく
3. そこでテーマ・構成を深掘り

---

## セッション例

```
Day 1: Brainstorm
- Author Profile を整理
- 12個のアイデアを出す
- /archive で保存

Day 2: Evaluate
- 6個をピックアップして評価
- 上位3つを Shortlist に
- /archive で保存

Day 3: Decide
- 最終比較
- 1つに決定
- book-contents-think へ移行
```

---

## よくある落とし穴

| 落とし穴 | 対策 |
|---------|------|
| 「自分に書く資格がない」と最初から諦める | 完璧な専門家である必要はない。「一歩先を行く先輩」でOK |
| 市場だけで選ぶ | 情熱がないと書ききれない。Passionを軽視しない |
| 既刊と被るから諦める | 同じテーマでも「あなたの視点」は唯一無二 |
| 決められない | 「どれも正解」と割り切る。書き始めれば見えてくる |
| 決めた後に迷う | Decision Log を見返す。理由を思い出す |
