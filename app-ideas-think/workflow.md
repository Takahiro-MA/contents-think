# App Ideas Brainstorming Workflow

「次に何のアプリを作るか」を決めるためのプロセス定義。

---

## 全体の流れ

```
[1. 発散] → [2. 評価] → [3. 決定] → [4. 深掘りへ移行]
Brainstorm    Evaluate     Decide      → app-contents-think
```

---

## Phase 1: 発散（Brainstorm Mode）

### 目的
とにかくアイデアを出す。質より量。

### やること
1. **制約の確認:** 時間、スキル、目的を `ideas-hub.md` に記入
2. **発散セッション:** 以下の切り口でアイデアを出す
   - 自分が困っていること
   - 周りの人が困っていること
   - 既存アプリへの不満
   - 「こんなのあったらいいな」
   - 学びたい技術から逆算
3. **記録:** 出たアイデアを Ideas Pool に追加（評価はしない）

### Tips
- 「これは無理」と思っても書く
- 組み合わせ・派生も歓迎
- 10個以上を目標に

---

## Phase 2: 評価（Evaluator Mode）

### 目的
アイデアを客観的に比較し、候補を絞る。

### 評価軸

| 軸 | 問い |
|----|------|
| **Feasibility** | 自分のスキル・時間で作れるか？ |
| **Market** | 誰が欲しがる？どれくらい切実？ |
| **Differentiation** | 既存と何が違う？ |
| **Personal Fit** | 自分がワクワクするか？強みを活かせるか？ |
| **Effort vs Impact** | 労力に見合うリターンがあるか？ |

### やること
1. 有望そうなアイデアを5-7個ピックアップ
2. 各アイデアを評価軸でスコアリング（H/M/L）
3. 比較表を作成
4. 上位3つを Shortlist に

---

## Phase 3: 決定（Decision Mode）

### 目的
「次に作るアプリ」を1つ決める。

### やること
1. Shortlist のアイデアを最終比較
2. 決定基準を明確化（今、何を優先するか？）
   - 学習目的なら → 技術的チャレンジ重視
   - 収益目的なら → Market重視
   - ポートフォリオなら → 見栄え・完成度重視
3. 決断し、Decision Log に記録
4. 選ばなかったアイデアは Parking Lot へ

### 決定を助ける問い
- 「半年後、どれを作っておけばよかったと後悔する？」
- 「どれなら、明日から作り始めたいと思える？」
- 「どれが一番、自分の成長につながる？」

---

## Phase 4: 移行

### やること
1. 決定したアイデアの概要をコピー
2. `app-contents-think` に新しいプロジェクトとして持っていく
3. そこで要件定義・設計を深掘り

---

## セッション例

```
Day 1: Brainstorm
- 制約を整理
- 15個のアイデアを出す
- /archive で保存

Day 2: Evaluate
- 7個をピックアップして評価
- 上位3つを Shortlist に
- /archive で保存

Day 3: Decide
- 最終比較
- 1つに決定
- app-contents-think へ移行
```

---

## よくある落とし穴

| 落とし穴 | 対策 |
|---------|------|
| 発散中に「これは無理」とフィルタリング | 評価は後。まず出す |
| 評価が感覚的になる | 評価軸を使って構造化 |
| 決められない | 「どれも正解」と割り切る。完璧な選択はない |
| 決めた後に迷う | Decision Log を見返す。理由を思い出す |
